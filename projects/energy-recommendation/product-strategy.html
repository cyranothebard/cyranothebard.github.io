Product Strategy

  • Primary users: grid planners and energy analysts; secondary: operations.
  • Value: reduced curtailment and energy waste; improved renewable integration.
  • Metrics: RMSE, MAPE, avoided curtailment, user adoption, SLA adherence.
  • Roadmap: pilot → scale to regions → API for partners → optimization add-ons.

Back to overview

User Personas
  • Grid Planner: plans day-ahead operations; needs scenario analysis and exportable forecasts.
  • Energy Analyst: validates model performance; needs feature attributions and backtests.
  • Operations: monitors live forecasts; needs reliable API/SLA and clear alerts.
KPIs
  • Forecast Accuracy: RMSE / MAPE by horizon; target ≤ previous baseline −10%.
  • Operational Impact: avoided curtailment (%), reduced peak deviations (%).
  • Reliability: API uptime ≥ 99.7%, P95 latency ≤ 200 ms.
  • Adoption: monthly active analyst users, # of automated recommendations accepted.
Deployment Model
  • Containerized FastAPI service with versioned model artifacts; /forecast and /explain endpoints.
  • Daily retraining job and drift monitoring; alerts route to on-call with playbooks.
  • Blue/green or canary updates to minimize operational risk.
Roadmap
  1. Pilot (single region): validate KPIs and user workflow.
  2. Scale-out: multi-region support and cost-optimized inference.
  3. Partner API: external consumers (DER, storage providers) via authenticated access.
  4. Optimization add-ons: recommender constraints, DR scheduling, storage dispatch.
